# تست Air780E با Arduino Uno

این اسکچ برای تست و عیب‌یابی ماژول Air780E از طریق Arduino Uno طراحی شده.

## اتصال سخت‌افزاری

```
Arduino Uno         Air780E
Pin 10 (RX)    ←   TXD
Pin 11 (TX)    →   RXD
GND            →   GND
3.3V یا 4.2V   →   VCC   (از 5V مستقیم استفاده نکنید)

اختیاری:
Pin 7          →   PWRKEY
Pin 8          ←   STATUS
```

## مرحله اول: تنظیم Baud Rate

Air780E پیش‌فرض **115200** baud دارد. SoftwareSerial روی Uno فقط تا ~57600 پایدار است.

**یک‌بار** این کار را انجام دهید:
1. در فایل `.ino`، مقدار `SET_BAUD` را به `1` تغییر دهید
2. آپلود کنید
3. در Serial Monitor بنویسید و Enter بزنید – ماژول به 9600 تنظیم می‌شود
4. `SET_BAUD` را دوباره به `0` برگردانید و دوباره آپلود کنید

## روش استفاده

1. کد را آپلود کنید
2. Serial Monitor را روی **115200 baud** باز کنید
3. `h` را تایپ و Enter بزنید برای دیدن راهنما

## ترتیب تست پیشنهادی

```
a  →  تست AT پایه (ماژول پاسخ می‌دهد؟)
a  →  CIMI نمایش داده می‌شود؟ (سیم کارت خوانده شد)
n  →  ثبت شبکه (صبر کنید تا "ثبت شبکه موفق" ببینید)
g  →  باز کردن GPRS
t  →  تست TCP connect
i  →  ارسال داده تست
d  →  قطع اتصال
```

## تفسیر AT+CSQ

```
AT+CSQ → +CSQ: xx,0
  0-5:  سیگنال خیلی ضعیف
  5-15: ضعیف
  15-25: متوسط (قابل قبول)
  25-31: خوب
  99:   بدون سیگنال
```

## تفاوت Air780 vs SIM800L

| | Air780E | SIM800L |
|--|---------|---------|
| شبکه | 4G LTE | 2G GPRS |
| ثبت | AT+CEREG? | AT+CREG? |
| GPRS فعال | AT+NETOPEN | AT+CIICR |
| TCP | AT+CIPOPEN | AT+CIPSTART |

## اگر ماژول پاسخ نمی‌دهد

1. **baud rate** – ممکن است هنوز 115200 باشد، SET_BAUD=1 را امتحان کنید
2. **ولتاژ** – حتماً 3.3V یا 4.2V (نه 5V)، جریان کافی
3. **سیم‌ها** – TX ماژول به RX آردوینو (پین 10)، نه برعکس
4. **PWRKEY** – اگر Pin 7 وصل است، دستور `p` را بزنید
5. **ریست** – ماژول را برق بزنید و دوباره امتحان کنید
